## 3. 스킬 화면

단순 기술 나열이 아니라, 프로젝트에서 해당 기술로 무엇을 구현했는지 중심으로 작성한다.

### Backend

**Spring Boot 3.x, Java 21, JPA, Spring Security Crypto, REST API**

- Spring Boot 3.5.x와 Java 21 기반으로 교통 AI 솔루션의 백엔드 API 서버를 구성했습니다.
- Controller, Service, Repository, DTO, Entity를 분리해 통행 이벤트, GPS 판정, 정산 후보, 관리자 대시보드, 회원/권한, 게시판 기능을 도메인 단위로 구현했습니다.
- JPA와 PostgreSQL을 연동해 차량 통행 이벤트, GPS 텔레메트리, Edge 상태, Ingress 상태, 정산 데이터, 감사 로그를 저장하고 조회할 수 있도록 구성했습니다.
- 로그인/회원가입 및 관리자 전용 대시보드 진입 구조를 구현해 사용자 역할에 따른 화면 접근 흐름을 설계했습니다.
- FastAPI Ingress에서 전달되는 차량 번호판/OCR 이벤트를 Spring Boot의 통행 판정 API로 연결하는 구조를 설계했습니다.

### Frontend

**Vue.js 3, Pinia, Vite, Axios, Vue Router, HTML5, CSS3, JavaScript**

- Vue.js 3와 Vite 기반으로 Hi-Five 서비스 소개 페이지, 로그인/회원가입, 게시판, 운영자 대시보드, 마스터 관리자 대시보드를 구현했습니다.
- Axios API 모듈을 분리해 운영자 요약, 통행 이벤트, 영상 상태, 장비 상태, GPS 판정, 정산 후보, 관리자 시스템 상태 데이터를 화면에 연결했습니다.
- Pinia 기반 인증 상태 관리와 Vue Router 가드를 활용해 로그인 여부 및 관리자 권한에 따라 접근 가능한 페이지를 분리했습니다.
- 대시보드 UI에서 KPI 카드, 실시간 YOLO 영상 패널, 최근 GPS 판정 테이블, 현장 알림, 장비 상태, 시스템 모니터링 정보를 구성했습니다.
- HTML/CSS 프로젝트에서는 GNB/LNB 메뉴, 서브 페이지 템플릿, 로그인/회원가입, 커뮤니티, 관광 정보 페이지를 공공기관형 레이아웃으로 구현했습니다.

### AI & Data

**Python, FastAPI, OpenCV, YOLO, OCR, TensorRT, Jetson Edge**

- FastAPI 기반 Ingress 서버를 구성해 Edge 단말에서 발생한 통행 이벤트와 번호판 인식 결과를 Spring Boot로 전달하는 API 어댑터를 구현했습니다.
- Jetson Edge 런타임 구조에서 YOLO 객체 인식, OCR 번호판 인식, 프레임 처리, 이벤트 생성, Spring Forwarder, Spool 처리 모듈을 분리했습니다.
- YOLO와 OCR 결과를 차량 통행 이벤트 및 GPS 장치 정보와 연결해 통행 판정/정산 로직으로 이어지는 AI 파이프라인을 구성했습니다.
- 실시간 영상 스트림과 운영 대시보드 화면을 연결해 AI 추론 결과가 사용자 화면에서 확인되도록 구성했습니다.
- 프로젝트 기준 확인 기술은 YOLO/OCR/OpenCV/FastAPI이며, PyTorch/TensorFlow 모델 학습 이력은 추가 자료가 있으면 별도 섹션으로 보강하는 것이 좋습니다.

### Database & Infra

**PostgreSQL, Docker, Jetson/Raspberry Pi 계열 Edge, Git/GitHub, Vite Build**

- PostgreSQL을 Spring Boot JPA와 연결해 통행, GPS, 정산, 장비, 회원, 게시판, 감사 로그 등 운영 데이터를 저장하는 구조를 구성했습니다.
- Dockerfile이 포함된 Spring Boot 백엔드 구조를 통해 배포 환경을 고려한 서버 구성을 준비했습니다.
- Jetson Edge와 FastAPI Ingress를 분리해 AI 추론/수집 계층과 비즈니스 처리 계층의 책임을 나누었습니다.
- Vue/Vite 기반 프론트엔드 빌드 구조를 사용해 서비스형 웹 화면과 쇼핑몰형 프론트엔드를 구성했습니다.
- Git/GitHub URL, 배포 URL, 시연 URL은 포트폴리오 공개 전 실제 링크로 교체해야 합니다.

### 상세보기

#### 문제 정의

기존 통행료 징수 시스템은 차량 인식, GPS 판정, 정산, 운영 관제 데이터가 각각 분리되기 쉽습니다.  
Hi-Five는 차량 번호판 OCR, GPS 기반 통행 위치, 운영 상태 데이터를 하나의 서비스 흐름으로 연결해 관제자가 실시간으로 확인하고 관리할 수 있는 스마트 톨링 시스템을 목표로 했습니다.

#### 핵심 기능

- Jetson Edge에서 차량/번호판 영상 처리 및 YOLO/OCR 추론
- FastAPI Ingress에서 Edge 이벤트 수신 및 Spring Boot 서버로 전달
- Spring Boot에서 통행 이벤트, GPS 텔레메트리, 정산 후보, 장비 상태, 관리자 데이터를 처리
- PostgreSQL에 통행 이력, GPS 판정, Edge 상태, Ingress 상태, 감사 로그 저장
- Vue.js 대시보드에서 실시간 영상, KPI, GPS 판정, 알림, 장비 상태, 정산 데이터를 시각화
- 관리자 권한 기반 대시보드 접근 제어

#### 구현 포인트

- 백엔드에서는 `controller`, `service`, `repository`, `dto`, `entity` 패키지를 분리해 도메인별 책임을 명확히 했습니다.
- FastAPI Ingress는 비즈니스 로직을 직접 처리하지 않고, Edge 이벤트를 Spring Boot로 전달하는 어댑터 역할로 설계했습니다.
- Vue 프론트엔드는 운영자 대시보드와 마스터 관리자 대시보드를 분리하고, Axios API 모듈을 통해 백엔드 데이터를 화면에 연결했습니다.
- 대시보드에는 YOLO 실시간 영상 패널, GPS 판정 테이블, 현장 알림, 장비 상태, 시스템 상태 카드 등을 구성했습니다.

#### 사용 기술과 역할

- Spring Boot: REST API, 통행/정산/회원/관리자 도메인 처리
- Java 21: 백엔드 비즈니스 로직 구현
- JPA/PostgreSQL: 운영 데이터 저장 및 조회
- FastAPI: Edge 이벤트 수신, 상태/메트릭 API, Spring Forwarding
- Vue.js 3: 서비스 소개 화면, 인증 화면, 운영자/관리자 대시보드
- Pinia/Vue Router: 인증 상태 관리, 관리자 접근 제어
- YOLO/OCR/OpenCV: 차량 및 번호판 인식 파이프라인
- Jetson Edge: 현장 영상 처리 및 AI 추론 실행 환경

#### 포트폴리오에서 강조할 문장

**AI 모델 결과가 실제 서비스에서 쓰이려면 API, 데이터베이스, 운영 화면까지 연결되어야 한다고 생각했습니다. 이 프로젝트에서는 Edge AI 추론 결과를 FastAPI와 Spring Boot를 거쳐 Vue 대시보드까지 연결하며 AI 풀스택 서비스의 전체 흐름을 구현했습니다.**

## 프로젝트 3. LUXE SHOP

### 카드 요약

- 프로젝트명: LUXE SHOP
- 종류: 개인 프로젝트 / 쇼핑몰 프론트엔드
- 기술: Vue.js, Vite, Bootstrap, HTML5, CSS3, JavaScript, JSON 데이터
- 한 줄 소개: 상품 목록, 카테고리, 상품 카드, 위시리스트, 빠른 추가 UI를 갖춘 패션 쇼핑몰 프론트엔드
- 산출물:
  - Vite 빌드 결과물
  - 상품 JSON 데이터
  - 상품 카드/카테고리/위시리스트/상세 UI 스타일
- 실제 작동 URL: [입력 필요]
- 로컬 경로: `개인 프로젝트/성남 문화관광 사이트/web03/index.html`

### 상세보기

#### 문제 정의

쇼핑몰 화면은 상품 탐색, 비교, 구매 전환 흐름이 빠르게 이어져야 합니다.  
LUXE SHOP은 상품 중심 UI에서 카드 레이아웃, 카테고리 필터, 빠른 추가, 위시리스트 같은 이커머스 기본 경험을 구현하는 것을 목표로 했습니다.

#### 핵심 기능

- 상품 JSON 기반 상품 목록 구성
- 의류, 액세서리, 신발, 가방 등 카테고리 기반 상품 분류
- 가격, 할인가, 별점, 리뷰 수, 재고, 배지 정보 표시
- 상품 카드 hover 효과, 상품 이미지 전환, 빠른 추가 버튼 구성
- 위시리스트 버튼 및 상품 상세 이미지 썸네일 UI 구성
- Bootstrap 기반 반응형 레이아웃과 커스텀 테마 컬러 적용

#### 구현 포인트

- Vue/Vite 빌드 구조를 사용해 컴포넌트 기반 쇼핑몰 프론트엔드를 구성했습니다.
- 상품 데이터는 JSON으로 분리해 UI와 데이터를 독립적으로 관리할 수 있도록 했습니다.
- 상품 카드, 카테고리 링크, 컬러 선택 버튼, 썸네일 상태 등 반복되는 쇼핑몰 UI 요소를 컴포넌트형 스타일로 구성했습니다.
- CSS 변수와 Bootstrap을 함께 사용해 고급 패션 쇼핑몰에 맞는 시각 톤을 구성했습니다.

#### 보완 필요

현재 폴더 기준으로는 쇼핑몰 백엔드와 PostgreSQL 연동 코드가 확인되지 않습니다.  
포트폴리오에서는 "쇼핑몰 프론트엔드 구현"으로 쓰는 것이 정확하며, Spring Boot/PostgreSQL 기반 주문·회원·장바구니 API가 있다면 별도 자료를 추가해 "풀스택 쇼핑몰"로 확장할 수 있습니다.

#### 포트폴리오에서 강조할 문장

**쇼핑몰 화면에서 중요한 것은 상품 정보를 빠르게 비교하고 구매 행동으로 이어지게 만드는 흐름이라고 보고, 상품 카드·카테고리·상세 UI·위시리스트 요소를 중심으로 프론트엔드를 구성했습니다.**
